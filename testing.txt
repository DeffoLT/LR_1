1)В якому статусі можуть перебувати файли при роботі з Git?
Comited; modified; staged.
2)Для чого використовується команда add?
Команда add додає вміст робочого кліента для наступног коміта. 
3)Для чого використовується команда status?
Git status показує стан файлів в робочій директорії.
4)Для чого використовується команда commit?
Бере всі дані, додані до індексу за допомогою git add, і зберігає їх у внутрішній базі даних.
5)Для чого використовується команда init?
Відкриваеться репозиторій.
6)Що необхідно писати в коментарях комітів?
Корисну інформацію, яка допоможе згадати дії.
7)Для чого використовується команда checkout?
Використовується для перемикання гілок та вивантаження їхнього вмісту в робочу директорію.
8)Для чого використовується команда reset?
Використовується в основному для скасування змін.
9)Для чого використовується команда log?
Перераховує коміти, зроблені в репозиторії.
10)Для чого використовується команда status?
Git status показує стан файлів в робочій директорії.
11)Як відновити кілька файлів з попередньо створеного коміта?
git reset
12)Як перейди до попереднього коміта по імені?
git log використовується для перегляду історії коммітів, починаючи з найсвіжішого та йдучи до витоків проекту.
13)Як створити нову гілку?
git branch - це свого роду "менеджер гілок". Вона вміє перераховувати ваші гілки, створювати нові, видаляти та перейменовувати їх.
14)Як перейти на нову гілку?
git checkout 
15)Як перейти на основну гілку?
Git checkout main
16)Як виконати злиття гілок?
git merge використовується для злиття однієї або декількох гілок у поточну.
